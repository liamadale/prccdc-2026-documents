PRCCDC Playbooks

Competition documentation for the Pacific Rim Collegiate Cyber Defense Competition (PRCCDC). All documents are designed to be printed — they render to PDF via Quarto + pdflatex. No internet access during competition.

See CONTEXT.md for full competition context, scoring rules, and writing style guidelines.

Building PDFs

Prerequisites

# Quarto CLI (renders .md and .qmd to PDF)
# https://quarto.org/docs/get-started/
quarto --version

# pdflatex (via TeX Live or MiKTeX)
pdflatex --version

# Required LaTeX packages (installed automatically by TeX Live on first render,
# or install manually via tlmgr):
#   fvextra    -- code block line wrapping
#   xurl       -- URL wrapping in PDF
#   enumitem   -- list formatting
#   longtable  -- multi-page tables
#   booktabs   -- table rules

On Ubuntu/Debian:

sudo apt install quarto texlive-full

On Windows: install Quarto and MiKTeX or TeX Live.

Rendering a Single Document

# From the playbooks/ directory:
quarto render incident-response/ir-playbook-linux.md
quarto render sec/sec-playbook.qmd

Output PDF is written next to the source file (e.g., ir-playbook-linux.pdf).

Rendering All Playbooks at Once

# From playbooks/ — renders every .md and .qmd with a pdf format block
for f in **/*.md **/*.qmd; do quarto render "$f"; done

Or use a shell one-liner that skips files without a YAML header:

find . -name "*.md" -o -name "*.qmd" | xargs -I{} quarto render {}

Inventory Sheets (LaTeX)

cd inventorysheets/
pdflatex prccdc-host-inventory.tex
pdflatex prccdc-host-enum-cheatsheet.tex
pdflatex linux-blueteam-cheatsheet.tex

Run pdflatex twice if cross-references or page numbers are wrong on first pass.

YAML Front Matter (Required on All Playbooks)

Every .md and .qmd playbook must start with this block (adjust title):

---
title: "Document Title"
format:
  pdf:
    toc: true
    toc-depth: 3
    pdf-engine: pdflatex
    geometry:
      - margin=0.75in
    fontsize: 9pt
    include-in-header:
      text: |
        \usepackage{fvextra}
        \DefineVerbatimEnvironment{Highlighting}{Verbatim}{breaklines,commandchars=\\\{\}}
        \usepackage{xurl}
        \usepackage{enumitem}
        \setlist[description]{style=nextline,leftmargin=0pt,labelindent=0pt,itemsep=2pt,parsep=0pt}
        \usepackage{longtable}
        \usepackage{booktabs}
---

fvextra and xurl are mandatory — without them, long code lines and URLs will overflow the page margin and become unreadable when printed.

Using Claude Code to Draft or Extend Playbooks

These playbooks were drafted with Claude Code, Anthropic’s CLI coding assistant. It has context about this project loaded via CONTEXT.md.

Workflow:

  1. Open this folder in VS Code (or any terminal).

  2. Launch Claude Code:

    claude

  3. Reference @CONTEXT.md at the start of a session so Claude understands the competition format, print constraints, and writing style.

  4. Ask Claude to draft, extend, or fix a specific playbook. Example prompts:

    • Draft a new playbook for hardening SSH on Linux. Follow the format in incident-response/ir-playbook-linux.md.
    • Add a section to service-ops-playbook-linux.md covering Apache service recovery.
    • Review sec/sec-playbook.qmd and check that all code blocks will wrap correctly when printed.

  5. After edits, render to verify the PDF looks correct:

    quarto render <file>

What Claude Code does well here: - Generating complete step-by-step procedures from a brief description. - Ensuring consistent formatting (tables, code blocks, verification steps) across all documents. - Catching bare markdown hyperlinks that will break in print (should be <https://...> or plain URLs). - Drafting inject response templates from a scenario description.

What to verify manually: - Commands tested against the actual distro/version in competition. - Any IP addresses, hostnames, or credentials are competition-specific placeholders. - PDF renders correctly before printing — check code block wrapping and table layout.
